# Releasing

ripsync ships cross-platform archives plus a Homebrew formula and a Windows MSI,
driven by [cargo-dist](https://opensource.axo.dev/cargo-dist/) and a `v*` tag.

## One-time setup

1. Install the tool: `cargo install cargo-dist` (binary name `dist`).
2. Create the Homebrew tap repository `beeboyd/homebrew-tap` (empty is fine;
   dist pushes the formula to it).
3. Add a `HOMEBREW_TAP_TOKEN` (or the dist-documented token) repo secret with
   push access to the tap.

## Configuration

All release settings live in `[workspace.metadata.dist]` in the root
`Cargo.toml`:

- `targets` — x86_64/aarch64 Linux (gnu), x86_64/aarch64 macOS, x86_64 Windows
  (msvc);
- `installers` — `shell`, `powershell`, `homebrew`, `msi`;
- `tap = "beeboyd/homebrew-tap"`, `publish-jobs = ["homebrew"]`;
- `include = ["dist/assets"]` attaches the man page and shell completions to
  every archive (generated by `cargo xtask dist-assets`);
- `cargo-auditable = true` embeds an SBOM in the release binaries.

After editing that block, regenerate the workflow:

```sh
dist init        # interactive; reads the metadata
dist generate    # writes .github/workflows/release.yml
```

The committed `release.yml` is a hand-authored stand-in (matrix build → man page
+ completions → archives with SHA-256 → GitHub Release). `dist generate`
replaces it with the canonical version that also runs the Homebrew and MSI
publish jobs.

## Cutting a release

1. Bump the workspace version in `Cargo.toml` and update `CHANGELOG.md`.
2. Commit, then tag and push:

   ```sh
   git tag v0.4.0
   git push origin v0.4.0
   ```

3. The tag triggers `release.yml`: it builds every target, generates the man
   page and completions, packages archives with checksums, and creates the
   GitHub Release. With `dist` wired, the Homebrew formula is pushed to
   `beeboyd/homebrew-tap` and the MSI is attached.

## Verifying

- `dist plan` previews the artifacts a tag would produce without building.
- Download an archive and confirm `ripsync --version`, `man ./assets/ripsync.1`,
  and the checksum match.
